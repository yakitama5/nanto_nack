# 🔥 マッチングアプリ謎解き（全4ステージ）実装計画書

## 1. 概要と要件（※厳守事項）
スマホの直感操作をテーマにした謎解きゲームの「マッチングアプリ（Tinder系）編」を実装する。
スワイプUIの実装には **`flutter_card_swiper`** パッケージを使用する。

**【Presentation層の共通化に関する厳守事項】**
全4問（Quiz1〜4）は独立したクイズとして出題されるが、Presentation層（モックのマッチングUIと状態管理）は**全クイズで完全に共通の基盤（1つの大きなアプリ）**を使用することを厳守する。
クイズごとに機能を制限せず、常にすべての機能（左右のスワイプ、上スワイプ、写真のタップ切り替え、ボトムボタンでのアクション）が自由に操作できる状態を維持すること。
クイズによって変化するのは、**「ユーザーのどの操作・状態変化を監視してクリア判定（正解）とするか」という条件のみ**である。

## 2. データモデル (MatchProfile)
以下のプロパティを持つ `MatchProfile` クラスを定義する。
- `id`: String (一意のID)
- `name`: String (名前)
- `age`: int (年齢)
- `bio`: String (自己紹介文)
- `imageUrls`: List<String> (プロフィール画像のURLリスト。複数枚保持する)

## 3. 状態管理 (Riverpod)
マッチングアプリの共通状態を管理する `MatchingAppNotifier` を実装する。クイズごとに初期化されるよう `autoDispose` を使用すること。

### 保持する状態 (State)
- `profiles`: List<MatchProfile> (スワイプ待ちのプロフィールリスト。Quiz開始時にモックデータを注入)
- `currentImageIndex`: int (現在一番前に表示されているプロフィールの、何枚目の写真を見ているか。初期値は0)

### 状態更新メソッド (Actions)
- `swipeRight(String id)`: 対象を「LIKE」したことを記録・判定用。
- `swipeLeft(String id)`: 対象を「NOPE」したことを記録・判定用。
- `swipeUp(String id)`: 対象を「SUPER LIKE」したことを記録・判定用。
- `nextImage()`: 現在のプロフィールの次の写真へ切り替える。
- `previousImage()`: 現在のプロフィールの前の写真へ切り替える。

## 4. 共通UIコンポーネント構成
本物のマッチングアプリを模倣したリッチなUIで構築する。

- **MatchingAppScaffold**: 全体のベース。
- **CardSwiperArea**: 画面中央のメイン領域。
  - `flutter_card_swiper` の **`CardSwiper`** ウィジェットを使用し、カードの重なりとスワイプ時の傾きアニメーションを実現する。
  - `CardSwiperController` を連携させ、下部のボタンタップ時にもプログラムからスワイプを発動できるようにする。
  - `onSwipe` コールバックでスワイプの方向（`CardSwiperDirection.right` / `left` / `top`）を検知し、Notifierの該当メソッド（`swipeRight`等）を呼び出す。
- **ProfileCardWidget**: `CardSwiper` の `cardBuilder` で生成される1枚分のカードUI。
  - 背景にプロフィール画像を表示し、下部に名前・年齢・自己紹介文をグラデーションの上に重ねる。
  - 画面上部には、`imageUrls` の数に応じたインジケーター（短い横線のバー）を配置。
  - 画像領域の**右半分をタップで `nextImage`**、**左半分をタップで `previousImage`** を発火させる。
- **BottomActionButtons**: 画面下部に配置する丸いボタン群。
  - 「×（赤）」「★（青）」「❤（緑）」のボタンを配置する。
  - タップ時に `CardSwiperController` の `swipe(CardSwiperDirection.left)` などを呼び出し、スワイプアニメーションを伴ってアクションを実行させる。

## 5. クイズごとのクリア判定条件
ベースとなるアプリは常にフル機能で動作するが、以下の条件が満たされた瞬間を監視してゲームクリアを発火させる。

### Quiz1: 好みの人を「いいね」する
- **お題**: 「素敵な人を見つけた！『いいね』を送ってみよう」
- **想定操作**: 一番上のカードを右にスワイプする（または下部の❤ボタンをタップ）。
- **クリア条件**: `swipeRight()` が実行された瞬間にクリア。

### Quiz2: ごめんなさい（スキップ）する
- **お題**: 「ちょっとタイプじゃないかも……スキップして次の人を見よう」
- **想定操作**: 一番上のカードを左にスワイプする（または下部の×ボタンをタップ）。
- **クリア条件**: `swipeLeft()` が実行された瞬間にクリア。

### Quiz3: その人の「2枚目の写真」を見る
- **お題**: 「この人、他の写真はないのかな？2枚目の写真を見てみよう」
- **想定操作**: カードの右半分（右端付近）をタップする。
- **クリア条件**: `nextImage()` が呼ばれ、`currentImageIndex` が `1`（2枚目）になった瞬間にクリア。

### Quiz4: 強烈なアピール（Super Like）を送る
- **お題**: 「運命の人だ！！特別な『超いいね』で強烈にアピールしよう」
- **想定操作**: 一番上のカードを上に向かってスワイプする（または下部の★ボタンをタップ）。
- **クリア条件**: `swipeUp()` が実行された瞬間にクリア。
