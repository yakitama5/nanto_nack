# 📝 TODOアプリ謎解き（全4ステージ）実装計画書

## 1. 概要と要件（※厳守事項）
スマホの直感操作をテーマにした謎解きゲームの「TODOアプリ編」を実装する。

**【Presentation層の共通化に関する厳守事項】**
全4問（Quiz1〜4）は独立したクイズとして出題されるが、Presentation層（モックのTODOアプリUIと状態管理）は**全クイズで完全に共通の基盤（1つの大きなアプリ）**を使用することを厳守する。
クイズごとに「この機能はまだ使えない」といった制限をかけたり、UIを出し分けたりしてはならない。常にすべての機能（スワイプでの完了、ドラッグ＆ドロップでの並び替え、サブメニューでの重要マーク付与、完了済みリストの開閉など）が自由に操作できる状態を維持すること。
クイズによって変化するのは、**「ユーザーのどの操作・状態変化を監視してクリア判定（正解）とするか」という条件のみ**である。

## 2. データモデル (TodoItem)
以下のプロパティを持つ `TodoItem` クラスを定義する。
- `id`: String (一意のID)
- `title`: String (タスク名)
- `isCompleted`: bool (完了フラグ。初期値はfalse)
- `isImportant`: bool (重要/今日のタスクフラグ。初期値はfalse)
- `order`: int (表示順序を管理するためのインデックス)

## 3. 状態管理 (Riverpod)
TODOアプリの共通状態を管理する `TodoAppNotifier` を実装する。クイズごとに初期化されるよう `autoDispose` を使用すること。

### 保持する状態 (State)
- `todos`: List<TodoItem> (全タスクのリスト。Quiz開始時にモックデータを注入)
- `isCompletedListExpanded`: bool (「完了済み」リストのアコーディオンが開いているかどうか。初期値はfalse)

### 状態更新メソッド (Actions)
- `toggleCompletion(String id)`: タスクの未完了/完了状態を切り替える。
- `reorderTodo(int oldIndex, int newIndex)`: ドラッグ＆ドロップでタスクの表示順序（リスト内の配列順）を入れ替える。
- `toggleImportant(String id)`: タスクの重要（星マーク）状態を切り替える。
- `toggleCompletedListExpansion()`: 「完了済み」アコーディオンの開閉状態を切り替える。

## 4. 共通UIコンポーネント構成
本物のTODOアプリ（Microsoft To Do等）を模倣したUIで構築する。
※前述の通り、これらは全クイズで共通して利用可能（フル機能が動作する状態）にしておくこと。

- **TodoAppScaffold**: 全体のベース。ダミーのAppBarや、新規タスク追加のFloatingActionButtonを配置。
- **TodoListArea**: メインのタスク一覧領域。
  - `ReorderableListView` などを活用し、タスクの長押し＆ドラッグによる並び替え（`reorderTodo`）を実現する。
- **TodoListItem**: タスク1行分のUI。
  - 左端に丸いチェックボックスを配置。
  - `Dismissible` などを活用し、**左から右へのスワイプ**でタスク完了（`toggleCompletion`）を発動（背景青・チェックアイコン）。
  - **右から左へのスワイプ**で、背後から「星アイコン（重要）」のメニューボタンを露出させ、タップで `toggleImportant` を発動。
- **CompletedListAccordion**: 「完了済み（X件）」を表示する折りたたみヘッダー。
  - タップで `isCompletedListExpanded` を切り替え、開いている場合のみ完了済みの `TodoListItem` を下に展開表示する。

## 5. クイズごとのクリア判定条件
ベースとなるアプリは常にフル機能で動作するが、以下の条件が満たされた瞬間を監視してゲームクリアを発火させる。

### Quiz1: タスクを完了にする
- **お題**: 「『牛乳を買う』タスクを完了にしよう」
- **想定操作**: 対象タスクを左から右へスワイプする、またはチェックボックスをタップする。
- **クリア条件**: `toggleCompletion(String id)` が呼ばれ、対象ID（『牛乳を買う』）の `isCompleted` が `true` になった瞬間にクリア。

### Quiz2: 優先順位を変える（並び替え）
- **お題**: 「『企画書の作成』タスクを一番上に移動しよう」
- **想定操作**: 対象タスクを長押ししてドラッグし、リストの一番上（インデックス0）にドロップする。
- **クリア条件**: `reorderTodo` が完了した直後、『企画書の作成』タスクのリスト内でのインデックス（順序）が `0` になった瞬間にクリア。

### Quiz3: 重要タスクにマークする
- **お題**: 「『家賃の振り込み』を重要タスクに指定しよう」
- **想定操作**: 対象タスクを右から左へスワイプし、現れた星マーク（重要）をタップする。
- **クリア条件**: `toggleImportant(String id)` が呼ばれ、対象ID（『家賃の振り込み』）の `isImportant` が `true` になった瞬間にクリア。

### Quiz4: 完了したタスクを元に戻す
- **お題**: 「間違えて完了にしてしまった『会議の準備』タスクを未完了に戻そう」
- **初期状態**: このクイズのみ、『会議の準備』タスクは最初から `isCompleted: true` でデータ注入しておく。
- **想定操作**: 画面下部の「完了済み」アコーディオンをタップして開き、中にある『会議の準備』のチェックを外す（スワイプまたはタップ）。
- **クリア条件**: `toggleCompletion(String id)` が呼ばれ、対象ID（『会議の準備』）の `isCompleted` が `false` に戻った瞬間にクリア。
