# 🚀 NantoNack 新規クイズカテゴリ 実装計画 (Additional Quizzes Implementation Plan)

本計画は、ショッピングカテゴリに続く5つの新カテゴリ（チャット、動画、マップ、アラーム、決済）を、`packages/quizzes/` 配下に順次実装していくためのステップバイステップのガイドラインです。

## 🎯 前提条件（AIアシスタントへの指示方針）
実装は一度にすべて行わず、**「1つのカテゴリが完全に動作し、テストを通過してから次のカテゴリに進む」**という直列のフロー（Phase 3の反復）で行います。

---

## Phase 1: クイズ共通基盤の拡張 (`packages/quiz_core`)
各カテゴリの開発に入る前に、全クイズで使い回す共通コンポーネントを準備します。

1. **[Step 1] 豪華なクリア演出 (Clear Overlay) の実装**
   * 正解アクション時に画面全体を覆う共通のオーバーレイWidgetを作成。
   * 波紋（Ripple）やコンフェッティ（紙吹雪）、カスタム言語の一瞬の日本語化など、NantoNack独自のアハ体験演出を実装。
   * 演出終了後、自動でリザルト画面に遷移するコールバック機能を持たせる。
2. **[Step 2] カスタム言語パーサーの統合**
   * 事前に定義した「カスタム言語仕様（JSONマッピング）」を `quiz_core` 内に実装し、`String` の拡張メソッド（例: `'Cart'.toAlienLanguage()`）としてどこからでも呼び出せるようにする。

---

## Phase 2: 新カテゴリのルーティングとメニュー統合 (`packages/app_main`)
ダッシュボード（ホーム画面）から、新しいクイズカテゴリを選べるようにします。

1. **[Step 1] ステージ選択UIの拡張**
   * ダッシュボードのリスト（またはグリッド）に、チャット、動画、マップ、アラーム、決済の5つの入り口（Card）を追加。
   * 現状は「未解放（🔒）」または「Coming Soon」状態にしておく。
2. **[Step 2] `go_router` のパス定義**
   * `/chat`, `/streaming`, `/map`, `/alarm`, `/payment` へのルーティング基盤を定義する。

---

## Phase 3: 個別カテゴリの開発サイクル (The Development Loop)
ここからが本番です。以下のStep 1〜5を、**1カテゴリ（例: チャット）ごとに**繰り返して実装します。

### 🔄 開発対象カテゴリ: 
（※ここに着手するカテゴリ名 `chat`, `streaming`, `map`, `alarm`, `payment` を当てはめて実行する）

1. **[Step 1: パッケージ構築とドメイン定義]**
   * `packages/quizzes/[カテゴリ名]` パッケージを作成し、`pubspec.yaml` を設定。
   * 対象カテゴリの仕様マークダウン（例: `chat.md`）を読み込み、4つのクイズ（Quiz 1〜4）のマスターデータとクリア条件を定義する。
   * Riverpodを用いて、対象アプリ特有の状態（例: `ChatState` でのメッセージリストや、`StreamingState` での動画再生状態）を管理するNotifierを作成。

2. **[Step 2: ベースUIとメンタルモデルの構築]**
   * 該当アプリ（LINE風、YouTube風など）の「あるある」なベースレイアウトを構築する。
   * テキストはすべて `toAlienLanguage()` を適用し、解読不能にする。
   * ※この段階ではまだロジックは繋がず、純粋なWidgetの見た目（Figma代わり）を完成させる。

3. **[Step 3: インタラクションと隠しジェスチャーの実装]**
   * タップ、長押し（Long Press）、スワイプ（Swipe to delete）、ドラッグなどのジェスチャー操作をUIに組み込む。
   * 動画のUIフェードアウトや、決済アプリの残高ブラー（`ImageFilter.blur`）など、カテゴリ特有のアニメーションを実装する。

4. **[Step 4: クイズロジック（判定）の結合]**
   * Quiz 1〜4 のお題（ミッション）に合わせて、ユーザーの操作がクリア条件を満たしたか判定するロジックをNotifier内に実装。
   * クリア条件を満たした瞬間、Phase 1で作った「豪華なクリア演出 (Clear Overlay)」を呼び出し、リザルト画面へ遷移させる。

5. **[Step 5: Testing (品質保証)]**
   * **Widget Test**: 長押しやスワイプなどのジェスチャーが正しく発火するかをテスト。
   * **Golden Test (`alchemist`)**: カスタム言語が適用されたUIのレイアウト崩れがないかを視覚的にテスト。
   * **E2E Test (`patrol`)**: 各クイズ（Quiz 1〜4）の開始からクリア、リザルト保存までの一連のフローを自動テスト。

---

## Phase 4: 全カテゴリ結合とポリッシュ
5つのカテゴリがすべて完成した後の最終調整です。

1. **[Step 1] ダッシュボードのロック解除**
   * Phase 2で追加した入り口のロックを外し、Driftのクリア状況に応じて順次解放されるように連携。
2. **[Step 2] 全体E2Eテストとパフォーマンス確認**
   * アプリ全体を通したPatrolテストを実行。
   * 画像アセット（ダミーの動画サムネイル等）のロード時間や、アニメーションの滑らかさ（Jankの有無）を確認・チューニングする。
