# ⚙️ NantoNack 設定画面 実装計画 (Settings Feature Implementation Plan)

本計画は、ダッシュボード画面の右上（⚙️アイコン）から遷移する、NantoNackの世界観を踏襲した設定画面を実装するための詳細なステップを定義します。

## Phase 1: 設定基盤と状態管理の準備 (`apps/app_main`, `packages/quiz_core`)

設定値の保存や外部連携に必要なパッケージと、状態管理を準備します。

1. **追加パッケージの導入:**
   * `pubspec.yaml` に `url_launcher`（ブラウザ起動）、`package_info_plus`（バージョン取得）、`shared_preferences`（軽量データ保存）を追加。
2. **状態管理 (Riverpod Notifier):**
   * `shared_preferences` を用いて「テーマ設定」と「UIスタイル設定」を保存・読み込み、アプリ全体（`MaterialApp`）に即座に反映させるための `SettingsNotifier` を構築。
3. **多言語化 (`slang`):**
   * 設定項目のテキスト、および初期化時の警告ダイアログの文言（「記憶を消去しますか？」等）を i18n 定義に追加し、`packages/quiz_core/lib/i18n/strings*.g.dart` を再生成。

## Phase 2: UIレイアウトと共通コンポーネントの構築

設定画面のベースとなる画面構成と、ポップな世界観を保つためのカードUIを作成します。

1. **画面遷移とベースレイアウト:**
   * `go_router` でダッシュボードから設定画面への全画面遷移 (`context.push('/settings')`) を定義。
   * `Scaffold` にシンプルな `AppBar`（タイトル「設定」と戻るボタン）を配置し、背景色はダッシュボードに馴染む薄いカラーに設定。
2. **カードデザインの実装:**
   * NantoNack風の角丸と軽いドロップシャドウを持つ、パステルカラーの `Card` ウィジェットベースを作成。
   * 各項目は `ListTile` を使用し、選択中の項目には `trailing: Icon(Icons.check)` が付くiOS設定アプリ風のUIコンポーネントを作成。

## Phase 3: 設定セクションごとの機能実装

各カード（意味的まとまり）ごとに具体的な機能を組み込みます。

1. **Card 1: アピアランス（見た目と操作感）:**
   * 背景色：薄い紫。
   * **テーマ**: 「システム設定 / ライトテーマ / ダークテーマ」のトグル実装。
   * **UIスタイル**: 「システム設定 / Android風 / iOS風」のトグル実装。
2. **Card 2: データとアカウント:**
   * 背景色：薄いオレンジ/ベージュ。
   * **購入の復元**: 課金状態のレシートを確認し、ローカルDB (`Drift`) の `is_premium` を更新する処理。
   * **プレイデータの初期化**: タイトルを赤色（`Colors.red`）で表示。タップ時に「これまでの学びが消え去ります…🥺」という遊び心のある警告ダイアログを表示。OKならDriftの `PlayLogs` と `StageRecords` を全削除する処理を実装。
3. **Card 3: アプリについて（リンク・ライセンス）:**
   * 背景色：薄い緑またはブルー系。
   * **外部リンク**: `url_launcher` を用い、開発者X、利用規約、お問い合わせを外部ブラウザで開く処理。
   * **ライセンス表示**: `package_info_plus` でアプリバージョンを取得し、`showLicensePage()` の上部にバージョン情報を含めて表示する処理。

## Phase 4: Testing（品質保証と自動テスト）

設定の永続化、ダイアログの挙動、およびUIの崩れに対するテスト設計です。

1. **Unit Test (`test`):**
   * `shared_preferences` をモック化し、テーマやUIスタイルの変更がNotifier経由で正しく保存・読み込みされるかをテスト。
2. **Widget Test (`flutter_test`):**
   * 初期化ボタンをタップした際に、警告ダイアログが正しく表示されるか（誤タップで即座に消えないか）のテスト。
3. **Golden Test (`alchemist`):**
   * ライトテーマ・ダークテーマそれぞれの状態で、設定画面のCardレイアウトやチェックマークの位置が崩れていないかの視覚的テスト。
4. **E2E Test (`patrol`):**
   * ダッシュボード右上の⚙️をタップ → テーマをダークに変更 → アプリについてを開く → データの初期化を実行し、ダイアログを承認 → ホームに戻りデータが消えているかを確認する一連のテスト。

------

## 🛠 実装の具体的なステップ

1. **[Setup]** `pubspec.yaml` に必要なパッケージを追加し、`slang` で多言語ファイルのテキスト定義を更新する。
2. **[Logic/Testing]** `SettingsNotifier` と `shared_preferences` の連携ロジックを作成し、**モックを用いたUnit Testを記述・通過させる**。
3. **[UI/Testing]** 設定画面のベースとなる `Scaffold` と、3つのカラフルな `Card` セクションを実装し、**`alchemist` でGolden Testを作成する**。
4. **[Integration]** 各ListTileのアクション（テーマ切り替え、`url_launcher` での外部ブラウザ遷移、ライセンス画面表示）を繋ぎ込む。
5. **[Integration/Logic]** データ初期化の「遊び心のあるダイアログ」を実装し、承認されたらDriftのRepositoryを呼び出してデータを全削除する処理を実装。
6. **[Testing]** `patrol` を用いて、ダッシュボードからの遷移・設定変更・データ削除ダイアログ操作を含むE2E Testを記述し、検証を完了させる。
