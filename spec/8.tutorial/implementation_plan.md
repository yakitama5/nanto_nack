# 👻 チュートリアル実装計画 (案内役：ナントム)

『NantoNack』の導入として、ナントムがユーザーを導き、自動で画面を操作しながらルールを説明する「体験型チュートリアル」の実装計画です。

## 概要

| 項目 | 内容 |
|------|------|
| パッケージ | `tutorial_coach_mark` |
| 案内キャラクター | ナントム (Nantom) |
| 表情パターン | 通常(Normal), 驚き(Surprised), 悲しい(Sad), 笑顔(Smile) |
| スコープ | 初回起動時 〜 Shoppingカテゴリ クイズ1完了まで |
| 特徴 | **スキップ機能搭載**、**システムによる自動画面遷移** |

---

## 🎨 ナントムの表情活用シーン

| 表情 | シーン | 意図 |
|------|------|------|
| **通常** | 挨拶、基本説明 | 落ち着いたトーンでの導入 |
| **驚き** | 文字化けの指摘、新機能発見 | ユーザーと一緒に驚き、異世界感を演出 |
| **悲しい** | 「読めない」ことへの共感 | 文字が読めない不便さを肯定する |
| **笑顔** | 自動遷移時、ミッション開始 | 期待感と楽しさを醸成する |

## 🛠️ チュートリアル・シナリオ（自動遷移フロー）

### Step 1: ホーム画面での挨拶 (Home)
1. **ウェルカム（笑顔）**
   * ターゲット：画面中央付近
   * テロップ：「ようこそ！ボクはナントム！」
2. **アプリの紹介（通常）**
   * ターゲット：変更なし (文字と表情だけ変更)
   * テロップ：「ここではボクが作ったアプリをならべてるよ」
3. **クイズへの導線（笑顔）**
   * ターゲット：プレイするボタン
   * テロップ：「一生懸命作ったからさっそく使ってみて！」
   * プレイする以外を押せない (スキップは可能)

### Step 2: カテゴリ選択の自動実行 (Home -> Shopping)
4. **自動遷移の予告（笑顔）**
   * ターゲット：Shoppingカテゴリのタイル
   * テロップ：「まずはショッピングアプリを体験してみよう。いくよ！」
   * ショッピング以外を押せない (スキップは可能)
   * ショッピングを選択したらチュートリアルの場合だけクイズ選択を飛ばしてクイズ1に飛ぶ

   * ターゲット：変更なし(表情とテロップだけ変更)

7. **操作のヒント（笑顔✨）**
   * ターゲット：変更なし(表情とテロップだけ変更)

## ⚙️ 技術的・UI仕様

### 1. スキップ機能
- `TutorialCoachMark` の標準機能を利用、またはカスタムオーバーレイとして「SKIP」ボタンを画面右上に常時配置。
- スキップ時は `shared_preferences` の `is_tutorial_completed` を直ちに `true` に更新し、全てのコーチマークを閉じる。

### 2. システムによる自動遷移の実装
- `TutorialCoachMark` の各 `TargetFocus` にある `identify` を利用。
- 特定のステップが終わったタイミングで、`context.pushNamed('/shopping')` などの遷移ロジックを発火させる。
- 遷移後の画面でも `TutorialCoachMark` を継続表示させるため、遷移先で次のターゲットを開始する連動ロジックを組む。

### 3. ナントムの配置レイアウト
- `ContentTarget` 内で `Row` を使用。
- `[ナントムの画像]` + `[吹き出し（テロップ）]` のセットを、スポットライト（強調範囲）の空いているスペース（上下左右）に `ContentAlign` で最適配置する。

---

## 💡 実装上の注意点
- **遷移待ちの制御**: 画面遷移のアニメーション中にコーチマークが表示されると位置がズレるため、`transition` が完了してから次のスポットライトを表示するよう `Future.delayed` 等で微調整する。
- **アセット管理**: ナントムの4パターンの画像は `assets/images/characters/nantom_*.png` として整理し、`TutorialStep` ごとに動的に切り替える。
