プロジェクトのコミットルールに従ってコミットしてください。

## コミット対象

$ARGUMENTS

（指定がない場合は現在のステージング済み変更または全変更をコミットします）

## コミットルール（必須）

### メッセージ形式

```
{label}: {message（変更内容 + Why）}
```

### ラベル一覧

| Label | 用途 |
|---|---|
| ✨feat | 新機能 |
| 🐛fix | バグ修正 |
| ♻️refactor | リファクタリング |
| 📚doc | ドキュメント |
| 🎨design | デザイン・UI/UX |
| 🐎perf | パフォーマンス |
| 🔧chore | ビルド・補助ツール・ライブラリ |
| 🚨test | テスト |
| 🗨️comment | コメント整理・TODO |
| ♻️generete | コード生成・Bootstrap |

### 例

```
✨feat: カート画面に合計金額を表示（ユーザーが購入前に総額を確認できるようにするため）
🐛fix: 割引アイテム含む場合に合計金額が二重計算される問題を修正
```

## 手順

1. `git diff --staged` または `git diff` で変更内容を確認する
2. 変更の種類を判断して適切なラベルを選ぶ
3. **「Why（なぜその変更が必要か）」** をメッセージに含める
4. 意味のある小さな単位でコミットを分割する（複数の変更を1コミットに混ぜない）
5. `git commit -m` でコミットを作成する
