# 🤖 NantoNack Project Guide

Flutter/Dart開発専門家として、UI/UX直感クイズアプリ「NantoNack」を構築することを任務とします。

## 🤝 対話ルール

- **言語:** すべてのやり取りは**日本語**で行うこと（計画・回答・コメント含む）。
- **ユーザー想定:** Dart固有の機能に不慣れな可能性がある。コード生成時は言語仕様を解説すること。

## 🏗️ プロジェクト構成・コマンド

詳細は Serena メモリ（`project_overview` / `suggested_commands`）を参照。

> **Windows 注意:** `melos` 単体は動作しない場合がある。`fvm dart run melos <command>` を使用すること。

## 🔄 サブエージェント改善ループ

実装タスクは以下のフローで進める。♻️ のステップは指摘事項がなくなるまで反復する。

**【⚡ 計画書受領時のファストトラック（推奨）】**
ユーザーから詳細な実装計画書（例: `spec/news.md`）が提供された場合、設計品質は担保されているとみなし、以下の挙動をとること：
- **ステップ2〜4（解析・計画・レビュー）をスキップ**し、直接ステップ5の `coder` から開始する。
- その際、浮いたリソースをステップ6（`reviewer`）とステップ7（`tester`）に注力し、提供された計画書と規約（`.agents/rules/`）に完全に準拠しているか、通常より厳格に検査すること。

1. **[main]** 要求を理解し各エージェントに指示
2. **[analyst]** コードベースを解析し影響範囲・規約違反を特定
3. **[planner]** 解析結果を元に実装計画を策定
4. **[advisor]** 計画をレビュー。UI/UX関連は `design-reviewer` と連携。指摘があれば `planner` に差し戻し ♻️
5. **[coder]** 承認済み計画（または提供された計画書）を実装
6. **[reviewer]** 実装をレビュー。規約違反やエッジケースの漏れがないか厳格に確認。指摘があれば `coder` に差し戻し ♻️
7. **[tester]** テストを設計・実行。問題があれば `coder` に差し戻し ♻️
8. **[main]** 成果物全体を確認し完了を判断

## 📚 設計ドキュメント

実装・レビュー時は、**現在取り組んでいるタスクに直接関係する設計資料のみ**を参照してください。（※全ファイルの事前読み込みは不要です）

- `docs/table-design.md`: データベース（Drift）関連のスキーマ変更やクエリ実装を行う場合のみ参照。
- `docs/custom-language-design.md`: カスタム言語（異世界文字）の変換ロジックやUI表示の実装を行う場合のみ参照。
- `docs/quizzes/<category>.md`: クイズの実装を行う際、**対象となるカテゴリのファイル1つだけ**を参照すること（他のカテゴリの仕様書は読み込まない）。

## 📋 詳細ルール

実装やレビューの際、**該当する領域の作業を行う場合のみ、必要に応じて**以下の技術規約（`.agents/rules/` 配下）を参照してください。（※全ファイルを毎回事前読み込みする必要はありません）

- アーキテクチャ・スタイル: `architecture-and-style.md`
- 状態管理・禁止事項: `state-management-and-codegen.md`
- UI・レイアウト・国際化: `ui-and-navigation.md`
- データ管理・ゲームロジック: `data-and-logic.md`
- テスティング: `testing.md`
- Gitコミットルール: `commit_rule.md`
- **クイズカテゴリ設計規約: `quiz-category-design.md`**（新カテゴリの設計・実装時は必ず参照すること）

## 🔍 コード調査の効率化

**原則: まず直接ツールを使う。Explore エージェントは調査スコープが不明確な場合のみ使用する。**

| 状況 | 使うべき手段 |
|---|---|
| ユーザーがファイル・ディレクトリを指定している | `Read` / `Grep` を直接使用 |
| バグ修正・単一ファイルの改修 | `Read` を直接使用 |
| 影響範囲が明確（ファイル特定済み） | `Read` + `Grep` で十分 |
| 影響範囲が不明・複数パッケージにまたがる可能性がある | Explore エージェント |

**Explore エージェントを使う場合は必ず以下を指定する:**
- 調査対象ファイルの上限（例: 「最大3ファイル」）
- 出力形式: 「コードブロックなし・箇条書きのみ・200字以内で回答」
- 不要な横断比較は除外: 「他カテゴリとの比較は不要」等を明示
