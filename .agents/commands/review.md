reviewer エージェントを使ってコードレビューを行ってください。

## レビュー対象

$ARGUMENTS

（指定がない場合は `git diff` または直近のコミットを対象にします）

## reviewer エージェントへの指示

以下の観点でレビューを行ってください:

### 1. プロジェクト規約の遵守（CRITICAL）

- `@riverpod`, `flutter_hooks`, `json_serializable` が含まれていないか
- Provider が手動で適切に定義されているか
- `copyWith` メソッドが手動実装されているか
- `DateTime.now()` が混入していないか（`clock.now()` を使用しているか）

### 2. Git コミットルールの検証

- ラベル（`✨feat`, `🐛fix` 等）が適切に使用されているか
- コミットの粒度が適切か（小さく意味のある単位）
- コミットメッセージに「Why（なぜ変更したか）」が含まれているか

### 3. 設計の妥当性

- 4層レイヤードアーキテクチャが守られているか
- `melos run lint` が通るか（警告ゼロか）

指摘は建設的に。良い実装は積極的に称賛してください。
